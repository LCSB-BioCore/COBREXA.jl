Exploring model contents

For practical reasons, COBREXA.jl supports many different model types. These comprise ones that reflect the storage formats (such as JSONModel and SBMLModel), and ones that are more easily accessible for users and mimic the usual workflows in COBRA methodology:

  • StandardModel, which contains and object-oriented representation of model internals, built out of Reaction, Metabolite and Gene structures, in a way similar to e.g. COBRApy
  • CoreModel, which contains array-oriented representation of the model structures, such as stoichiometry matrix and the bounds vector, in a way similar to e.g. COBRA toolbox

The fields in StandardModel structure can be discovered using fieldnames as follows:

using COBREXA

fieldnames(StandardModel)

!isfile("e_coli_core.json") &&
    download("http://bigg.ucsd.edu/static/models/e_coli_core.json", "e_coli_core.json");

sm = load_model(StandardModel, "e_coli_core.json")
typeof(sm.reactions)

fieldnames(Reaction)
(:id, :name, :metabolites, :lb, :ub, :grr, :subsystem, :notes, :annotations, :objective_coefficient)

This process (along with e.g. Tab completion in REPL) allows you to pick various information about many objects, for example about a specific reaction:

sm.reactions["TALA"].name
"Transaldolase"
sm.reactions["TALA"].grr #gene-reaction relationship
2-element Vector{Vector{String}}:
 ["b2464"]
 ["b0008"]
sm.reactions["TALA"].subsystem
"Pentose Phosphate Pathway"
sm.reactions["TALA"].ub #upper rate bound
1000.0
